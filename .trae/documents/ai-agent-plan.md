# AI 操作电脑系统功能计划

## 一、功能概述

允许 AI 助手在用户授权的情况下执行电脑系统操作，如文件管理、应用程序控制、系统设置等。功能通过开关控制，确保用户有完全的控制权。

## 二、安全设计原则

### 2.1 分层权限模型
- **级别 0**：禁用（默认）- AI 无法执行任何系统操作
- **级别 1**：低风险操作 - 文件读取、打开网页、启动应用
- **级别 2**：中等风险操作 - 文件写入/创建、剪贴板操作
- **级别 3**：高风险操作 - 文件删除、系统命令执行（需二次确认）

### 2.2 操作确认机制
- 高风险操作必须弹出确认对话框
- 显示操作详情，用户可选择"执行"、"拒绝"或"始终允许"
- 支持"信任此操作类型"白名单

### 2.3 审计日志
- 记录所有 AI 发起的系统操作
- 包含时间、操作类型、参数、结果
- 用户可查看历史记录

## 三、功能范围定义

### 3.1 文件操作
| 操作 | 风险级别 | 说明 |
|------|----------|------|
| 读取文件 | 低 | 读取文本文件内容 |
| 创建文件 | 中 | 在指定路径创建新文件 |
| 写入文件 | 中 | 修改现有文件内容 |
| 删除文件 | 高 | 删除指定文件（需确认） |
| 创建文件夹 | 中 | 创建目录 |
| 列出目录 | 低 | 获取目录内容列表 |
| 移动/重命名 | 中 | 移动或重命名文件 |

### 3.2 应用程序操作
| 操作 | 风险级别 | 说明 |
|------|----------|------|
| 启动应用 | 低 | 打开已安装的应用程序 |
| 关闭应用 | 中 | 关闭指定进程 |
| 打开URL | 低 | 在浏览器中打开链接 |

### 3.3 剪贴板操作
| 操作 | 风险级别 | 说明 |
|------|----------|------|
| 读取剪贴板 | 中 | 获取剪贴板文本 |
| 写入剪贴板 | 低 | 设置剪贴板内容 |

### 3.4 系统信息
| 操作 | 风险级别 | 说明 |
|------|----------|------|
| 获取系统信息 | 低 | CPU、内存、磁盘使用情况 |
| 获取进程列表 | 低 | 当前运行的应用程序 |
| 截图 | 中 | 获取屏幕截图 |

### 3.5 系统命令（受限）
| 操作 | 风险级别 | 说明 |
|------|----------|------|
| 执行命令 | 高 | 仅限预定义的安全命令白名单 |

## 四、技术架构

### 4.1 前端模块 (src/js/modules/ai-agent.js)
```javascript
// AI Agent 模块
- 权限状态管理
- 操作请求发送
- 确认对话框 UI
- 审计日志展示
```

### 4.2 后端处理器 (main/ai-agent.js)
```javascript
// IPC 处理器
- ai-agent-execute: 执行系统操作
- ai-agent-get-permissions: 获取权限配置
- ai-agent-set-permissions: 设置权限配置
- ai-agent-get-log: 获取操作日志
```

### 4.3 安全沙箱
- 路径限制：仅允许访问用户指定的目录
- 命令白名单：仅允许预定义的安全命令
- 文件类型限制：禁止访问敏感系统文件

## 五、UI 设计

### 5.1 设置界面
在 AI 设置抽屉中添加"系统操作"部分：
- 总开关：启用/禁用 AI 系统操作
- 权限级别选择：0-3 级
- 允许的目录列表：用户可添加允许 AI 访问的文件夹
- 操作日志入口：查看历史操作记录

### 5.2 确认对话框
```
┌─────────────────────────────────────────┐
│  ⚠️ AI 请求执行系统操作                    │
├─────────────────────────────────────────┤
│  操作类型：删除文件                        │
│  目标路径：C:\Users\xxx\Documents\test.txt │
│                                          │
│  [执行] [拒绝] [始终允许此类操作]           │
└─────────────────────────────────────────┘
```

### 5.3 对话界面提示
- 当 AI 想要执行系统操作时，消息中显示操作卡片
- 用户可点击卡片查看详情并确认

## 六、实现步骤

### 第一阶段：基础架构
1. 创建 `main/ai-agent.js` 后端模块
2. 创建 `src/js/modules/ai-agent.js` 前端模块
3. 在 `preload.js` 中暴露 API
4. 在 `ipc-handlers.js` 中注册处理器

### 第二阶段：权限系统
1. 实现权限配置存储（localStorage + 加密存储敏感数据）
2. 实现权限级别检查逻辑
3. 实现操作确认对话框 UI
4. 实现审计日志记录

### 第三阶段：操作实现
1. 实现文件操作 API
2. 实现应用程序操作 API
3. 实现剪贴板操作 API
4. 实现系统信息获取 API

### 第四阶段：AI 集成
1. 修改 `ai-chat.js` 支持工具调用（Function Calling）
2. 定义系统操作工具 schema
3. 实现 AI 响应解析与操作执行
4. 实现操作结果反馈给 AI

### 第五阶段：安全加固
1. 实现路径访问限制
2. 实现命令白名单
3. 实现敏感文件保护
4. 添加操作频率限制

## 九、风险提示

1. **数据丢失风险**：AI 可能误删重要文件
2. **隐私风险**：AI 可能读取敏感信息
3. **系统稳定性**：AI 可能执行影响系统的操作
4. **安全风险**：恶意提示词可能利用 AI 执行危险操作

## 十、缓解措施

1. 默认禁用，需用户主动开启
2. 高风险操作强制确认
3. 详细的操作日志
4. 路径和操作类型限制
5. 用户可随时撤销权限
6. 敏感文件扩展名保护（.env, .key, .pem 等）
